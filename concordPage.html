Concord

Description

A communication platform for everyone that provides a place where people can reach a consensus.

proxy pattern through our Server class, we ensure that clients can easily communicate with the “backend” of our system through RMI. Furthermore, through the login() methods, the Server ensures that only clients that provide a valid username and password can talk to the backend and receive a User object to be associated with. This feature of associating a client with a user also ensures that any client only has the permissions that their associated user does.

every user has an ArrayList of Invitations from other users, which they can then use to join a new group if they so choose
